# 遇见成都 —— 静态网页项目介绍

## 一、项目概述

本项目是一个以"成都旅游"为主题的静态展示网页，采用纯 HTML + CSS + 原生 JavaScript 构建，无需任何第三方框架或依赖。页面通过精美的视觉设计和流畅的交互体验，向访客展示成都的特色文化、热门景点和出行建议。

## 三、页面结构与功能说明

### 3.1 导航栏（Navbar）

- **位置**：页面顶部，始终可见
- **功能**：
  - 提供"首页 / 特色体验 / 热门景点 / 出行建议"四个锚点链接
  - 滚动超过 40px 后自动切换为毛玻璃半透明背景（`.scrolled` 样式）
  - 根据当前滚动位置自动高亮对应导航项（`.active`）

**实现方式**：
- 使用 `position: sticky` + `backdrop-filter: blur()` 实现吸顶毛玻璃效果
- 通过 `scroll` 事件监听，根据各 `section` 与视口的相对位置计算当前应高亮的导航项
- 点击导航项时，使用 `window.scrollTo({ behavior: 'smooth' })` 平滑滚动到目标区域，并通过 `history.replaceState` 更新 URL hash

### 3.2 首屏横幅（Hero Banner）

- **内容**：渐变背景 + 标题 + 副标题 + 描述 + CTA 按钮
- **视觉**：多色渐变叠加半透明遮罩，营造氛围感

**核心交互 —— 打字机效果**：
- 主标题"来成都，感受慢生活与烟火气"在页面加载后逐字显示
- 使用 `setTimeout` 递归调用，每 80ms 显示一个字符
- 实现前先将标题文字清空并设为 `visibility: visible`，逐字符追加回 DOM

```javascript
const type = () => {
  if (i < text.length) {
    h1.textContent += text[i++];
    setTimeout(type, 80);
  }
};
```

### 3.3 向下滚动提示

- 底部居中的"向下滚动"文字 + 箭头图标
- 箭头使用 CSS `@keyframes bounce` 实现上下弹跳动画
- 整体以 `fadeUp` 动画在 1 秒后淡入显示

### 3.4 特色体验推荐（Food Section）

- 三栏网格布局，展示三项成都特色体验：
  - 川剧变脸
  - 鹤鸣盖碗茶
  - 深夜火锅
- 每张卡片包含 emoji 图标、标题、描述和建议标签（`.exp-tip`）

### 3.5 热门景点推荐（Spots Section）

- 三列两行共 6 张景点卡片，使用 CSS Grid 布局
- 每张卡片包含：
  - 背景图片（通过 CSS `background-image` 设置，hover 时放大 1.05 倍）
  - 半透明遮罩层
  - 景点名称、简介、星级评分

**图片处理**：
- 所有图片通过 `.card-image` 类设置 `background-size: cover; background-position: center` 保证比例一致
- 同时设置了 fallback 背景色，图片加载失败或离线时仍有视觉占位

### 3.6 出行建议（Travel Tips）

- 三栏卡片布局
- 内容涵盖：最佳季节、交通方式、必吃推荐
- 卡片使用白色背景 + 圆角 + 阴影，hover 时上浮

### 3.7 滚动淡入动画（Scroll Fade-In）

页面中所有带 `.fade-in` 类的元素在进入视口时自动触发淡入 + 上移动画。

**实现方式**——使用 **Intersection Observer API**：

1. 创建 `IntersectionObserver`，设置 `threshold: 0.2`（元素 20% 可见时触发）
2. 遍历所有 `.fade-in` 元素，为每个元素设置 `data-delay`（按所在位置对 3 取模 ×100ms，实现错落入场）
3. 当元素进入视口时，添加 `.visible` 类，触发 CSS transition

```css
.fade-in {
  opacity: 0;
  transform: translateY(22px) scale(0.96);
  transition: opacity 0.55s ease, transform 0.55s ease;
}
.fade-in.visible {
  opacity: 1;
  transform: translateY(0) scale(1);
}
```

### 3.8 滚动性能优化

- 使用 `requestAnimationFrame` 包装 scroll 事件处理，避免频繁触发导致的性能问题
- scroll 事件设置 `{ passive: true }`，不阻塞默认滚动行为
- 使用 `ticking` 标志位防止同一帧内重复执行

```javascript
let ticking = false;
const handleScroll = () => {
  if (ticking) return;
  ticking = true;
  requestAnimationFrame(() => {
    // ... 实际逻辑
    ticking = false;
  });
};
```

### 3.9 响应式设计

通过 CSS Media Queries 实现三档适配：

| 断点 | 布局调整 |
|------|----------|
| ≤900px | 三列网格变为两列；导航栏垂直排列 |
| ≤600px | 所有网格变为单列；导航链接换行居中；隐藏滚动提示 |
